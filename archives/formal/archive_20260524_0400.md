已修改文件列表：
- uni_agent/trainer/framework/framework.py：跳过 RewardLoopWorker + 移除 debug 日志和默认值

核心修改内容：

1. **跳过 RewardLoopWorker，使用 in-process reward 直传**
   - agent_runner 通过 evaluate_in_env 在 Docker 容器内计算 reward，
     然后通过 complete_session(reward_info=...) 传回 framework
   - 之前 framework 仍调用 RewardLoopWorker.compute_score() 二次计算 reward，
     但 RewardLoopWorker 缺少 Docker env 上下文且数据集缺少 verl 标准字段
   - 修复：当 session_trajectories 已有 reward_info 时，直接使用 in-process reward，
     不再走 RewardLoopWorker
   - 同时避免了 compute_score 返回格式不匹配、data_source/ground_truth 缺失等兼容问题

2. **移除不再需要的 debug 日志和默认值**
   - 移除 [TQ_DEBUG] 前缀日志（EXCEPTION/EMPTY_TRAJECTORIES/SUCCESS）
   - 移除 _trajectory_to_reward_dataproto 中 data_source/reward_model 的默认值填充
   - 这些都是走 RewardLoopWorker 路径时的临时修复，现在不再需要

3. **Qwen3-4B vs Qwen3.5-4B FSDP 兼容性**
   - Qwen3.5-4B 在 param_offload=true 时 FSDP offload 报 AssertionError
     (flat_param.data.data_ptr() != flat_param._local_shard.data_ptr())
   - param_offload=false 时 cuDNN 报 CUDNN_STATUS_NOT_INITIALIZED（GPU 内存不足）
   - gpu_memory_utilization=0.25 时 vLLM 无法分配 KV cache
   - Qwen3-4B (Qwen3-4B-Instruct-2507) 在同样配置下无此问题，训练完整跑通
   - 结论：Qwen3.5-4B 的 FSDP 参数布局与 offload 模式不兼容，需要 verl 侧修复
