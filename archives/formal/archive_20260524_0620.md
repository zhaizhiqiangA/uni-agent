排查并解决 cuDNN conv3d 初始化失败和 FSDP offload 不兼容问题。

- examples/swe_agent_blackbox/config/swe_agent_blackbox.yaml：训练配置适配 Qwen3.5-4B
- 容器内 cuDNN：从 cudnn9-cuda-13 (9.17.1) 替换为 cudnn9-cuda-12-9 (9.19.0.56)

Git Commit 信息：

commit 描述：fix(config): adapt training config for Qwen3.5-4B compatibility

核心修改内容（从上一次正式存档 archive_20260524_0400 至今）：

1. **训练配置适配 Qwen3.5-4B**
   - tensor_model_parallel_size: 4→8，减少单卡 vLLM 推理显存占用
   - gpu_memory_utilization: 0.5→0.3，给 FSDP 和 cuDNN 留更多空间
   - enable_sleep_mode: false→true，vLLM 在训练阶段释放 GPU 显存
   - max_assistant_turns: 2→1，单轮交互验证

2. **cuDNN conv3d CUDNN_STATUS_NOT_INITIALIZED 根因分析与修复**
   - 现象：Qwen3.5-4B 是 VL 模型，包含 patch_embed（Conv3d），
     训练时 FSDP forward 调用 F.conv3d 触发 cuDNN 初始化失败
   - 根因：容器安装的是 cudnn9-cuda-13 (9.17.1.4)，与 PyTorch 2.10.0+cu129
     的 CUDA 12.9 runtime 版本不匹配
   - 推理不受影响：vLLM 的 Conv3dLayer 使用 _forward_mulmat（F.linear 重写）
     绕过 cuDNN，不走原生 F.conv3d
   - 修复：卸载 cudnn9-cuda-13，安装 cudnn9-cuda-12-9 (9.19.0.56)
   - 验证：单卡 conv3d 测试通过，训练 4 步无报错

3. **FSDP offload AssertionError（已绕过，未根修）**
   - 现象：param_offload=true 时 offload_fsdp_model_to_cpu 断言
     flat_param.data.data_ptr() != flat_param._local_shard.data_ptr()
   - 原因：Qwen3.5-4B 的 FSDP 参数布局在 unflatten/reshard 后
     flat_param.data 被重新分配到不同显存地址
   - 当前方案：enable_sleep_mode=true 配合 param_offload=true，
     通过 sleep mode 管理 GPU 显存切换，避免 AssertionError 触发
   - 训练 4 步验证通过，未再出现此错误

4. **val_before_train 阶段容器超时问题（未修复）**
   - swe-bench 评测容器启动时 pip install swe-rex 网络慢导致 600s 超时
   - 目前通过 val_before_train=false 跳过 val 阶段绕过
   - 正式归档时已恢复 val_before_train=true

5. **中间尝试过的方案（已回退）**
   - sitecustomize.py 全局禁用 cuDNN：已删除，替换为正确版本 cuDNN
   - param_offload=false：导致 cuDNN workspace 不足，已恢复为 true
   - gpu_memory_utilization=0.25：vLLM 无法分配 KV cache，已恢复为 0.3
