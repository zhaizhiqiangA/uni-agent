本次存档目的：
为 SWEAgentFramework 新增 Ray remote 子进程执行 agent_runner 的能力，解决 agent_runner 中阻塞操作（sleep、同步 I/O 等）阻塞框架 event loop 的问题，支持稳定的并发 session 执行。

已修改文件列表：
- examples/swe_agent_blackbox/subprocess_runner.py：新增 Ray remote 函数 + Stub SessionHandle/Runtime
- examples/swe_agent_blackbox/framework.py：SWEAgentFramework 重载 _run_session，通过 Ray worker 执行 agent_runner

Git Commit 信息：
commit 号：2986436
commit 描述：feat(swe_agent_blackbox): run agent_runner in Ray worker process — Override SWEAgentFramework._run_session to execute agent_runner via @ray.remote, isolating blocking ops from event loop. Verified with max_concurrent_sessions=2 and =8.

核心修改内容：

1. **新增 subprocess_runner.py**（104 行）
   - `_StubSessionHandle`：dataclass，提供真实的 session_id 和 base_url，让 agent_runner 能正常访问 Gateway 做 LLM 调用
   - `_StubSessionRuntime`：拦截 `complete_session` 捕获 reward_info，其余方法为 no-op
   - `remote_agent_run`：`@ray.remote(num_cpus=0)` 函数，在独立 Ray worker 进程中运行 agent_runner，内部用 `asyncio.run()` 驱动 async agent_runner

2. **修改 framework.py**
   - `_resolve_runner()`：从 `self.agent_runner`（可能是 functools.partial）提取 FQN 字符串 + kwargs，避免序列化 partial 闭包
   - `_run_session()` 重载：
     - `create_session` 在父进程（不变）
     - `await self.agent_runner(...)` 替换为 `ray.get(remote_agent_run.remote(...))`
     - 父进程拿到 reward_info 后调真正的 `complete_session(reward_info=...)`
     - `finalize_session` 在父进程（不变），Gateway 自然收集 trajectories
     - `_score_trajectories` 逻辑不变

3. **设计要点**
   - agent_runner（uniagent/mini_swe/未来其他）完全不需要改动
   - Session 生命周期控制在父进程，子进程只做 agent 执行的"体力活"
   - 并发由父进程 Semaphore 控制，Ray worker 按需创建（num_cpus=0，纯 I/O）
   - 错误传播由 Ray 自动处理，走到已有的 `except → abort_session` 分支

4. **验证结果**
   - max_concurrent_sessions=2：4 step 完成，exit 0
   - max_concurrent_sessions=8：9 个独立 Ray worker 并行执行，4 step 完成，exit 0
   - 训练循环（rollout → log_prob → adv → update_actor → checkpoint）正常闭环
