修复训练配置问题，使 SWE-agent blackbox recipe 能在训练模式下正常启动。
同时改进推理脚本的并发控制和框架的错误日志输出。

已修改文件列表：

- examples/swe_agent_blackbox/parallel_infer.py：max_concurrent_sessions 从 0 改为 2
- examples/swe_agent_blackbox/scripts/run_train.sh：改用 main_ppo_sync 替代 main_ppo
- uni_agent/trainer/framework/framework.py：_run_batch_to_tq 增加 prompt 级别失败日志

环境依赖修复（非代码变更）：
- accelerate 1.12.0 → 1.13.0（修复 Parameter.__new__() _is_hf_initialized 兼容问题）
- 新增 TransferQueue==0.1.7（main_ppo_sync 强制依赖）

核心修改内容：

1. **swe_agent_blackbox.yaml 训练配置修复（3 处）**
   - `model.override_config.attn_implementation: eager`：actor FSDP 加载模型时默认使用 flash_attention_2，
     但环境未安装 flash-attn，添加 override 强制用 eager
   - `reward.custom_reward_function.path`：从 `examples.swe_agent_blackbox.reward` 改为
     `pkg://examples.swe_agent_blackbox.reward`，使 verl 的 load_extern_object 通过
     importlib.import_module 加载而非文件路径查找（Ray worker 工作目录不含项目根）
   - `TRANSFER_QUEUE_ENABLE: null` → `""`：Ray runtime_env.env_vars 不接受 None 值，
     必须是 Dict[str, str]

2. **run_train.sh 改用 main_ppo_sync**
   - 根因：旧版 main_ppo（ray_trainer.py）的 AgentLoopManager.create() 不传 replay_buffer，
     导致 AgentFrameworkRolloutAdapter.create() 的 assert replay_buffer is not None 失败
   - 新版 main_ppo_sync.py 在调用时正确传递 replay_buffer=self.replay_buffer
   - 注意：main_ppo_sync 强制依赖 TransferQueue

3. **parallel_infer.py max_concurrent_sessions**
   - 从 0（无限制）改为 2，避免同时启动过多 Docker 容器和 LLM session 导致 OOM
   - 训练配置 swe_agent_blackbox.yaml 中已有 max_concurrent_sessions: 2，推理侧同步

4. **framework.py 失败日志增强**
   - _run_batch_to_tq 中 asyncio.gather(return_exceptions=True) 捕获的异常原先只记录短消息
   - 改为 logger.error("prompt %d failed: %s", i, reason, exc_info=outcome) 输出完整堆栈
   - 便于排查训练模式下 "All rollouts failed" 的具体原因
