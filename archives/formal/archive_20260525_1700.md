核心修改内容（从上一次正式存档 archive_20260524_1302 至今）：

1. **删除冗余 _detect_tool_parser**
   - agent_runner.py 和 parallel_infer.py 中有完全相同的 _detect_tool_parser 函数
   - 该函数通过读取 tokenizer_config.json 的 chat_template 检测 tool parser 类型
   - 训练场景：gateway 的 parser 由 swe_agent_blackbox.yaml 的 multi_turn.format 硬编码；agent_runner 的 parser 从 agent_config.yaml 读取
   - 推理场景：parallel_infer.py 已有 --tool-parser CLI 参数
   - 自动检测逻辑脆弱（只靠字符串匹配），无实际必要
   - 改动：agent_config.yaml 新增 tool_parser: qwen3_coder；agent_runner 简化为三选一 fallback；parallel_infer 直接用 CLI 参数

2. **SWEBenchDataset 注入 verl 标准 reward 字段**
   - 背景：NaiveRewardManager.run_single() 硬访问 non_tensor_batch["data_source"] 和 non_tensor_batch["reward_model"]["ground_truth"]
   - 当前数据集（swe_bench_verified、swe_rebench、r2e_gym）缺少这两个字段
   - 新增 SWEBenchDataset(RLHFDataset)，覆写 __getitem__ 从 extra_info.tools_kwargs.reward 提取并注入
   - data_source 取 reward.name（如 "swe_bench"），reward_model 为 {"ground_truth": {}}（占位）
   - 训练侧：swe_agent_blackbox.yaml 的 data.custom_cls 配置指向 SWEBenchDataset
   - 推理侧：parallel_infer.py 的 _inject_reward_fields 实现相同逻辑，TensorDict 构建时同步写入
   - 目的：为后续恢复 compute_score 路径（而非 in-process reward 直传）做准备

3. **_allocate_port 端口可用性验证**
   - 原实现：纯 counter 递增，不验证端口是否被占用
   - 改进：counter + socket.bind 验证，OCCUPIED 时跳过继续递增
   - 解决并发 session 端口冲突问题

4. **parallel_infer.py 冗余清理**
   - 删除 load_agent_config 调用（加载后变量 agent_cfg 从未使用，agent_runner 自己会重新加载）
   - 删除对应的 import
