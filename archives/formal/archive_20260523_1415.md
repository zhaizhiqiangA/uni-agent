本次存档目的：
修复训练模式下 agent_runner 的容器创建和运行时问题，
包括 kwargs 兼容、端口竞态、swerex 缺失、tool_parser 缺失等。

已修改文件列表：
- examples/swe_agent_blackbox/agent_runner.py：容器名唯一化、端口分配、swerex 自动安装、**kwargs 兼容
- examples/swe_agent_blackbox/mini_swe_agent_runner.py：**kwargs 兼容
- examples/swe_agent_blackbox/config/swe_agent_blackbox.yaml：添加 multi_turn.format tool_parser 配置
- uni_agent/trainer/framework/framework.py：临时 debug 日志（已移除）

核心修改内容：

1. **agent_runner **kwargs 兼容**
   - framework.py _run_prompt_sessions_to_tq 从 sample_fields 提取 "agent_name" 传入 runner_kwargs
   - swe_agent_runner 和 mini_swe_agent_runner 的签名不接受 agent_name，导致 TypeError
   - 两个 runner 均添加 **kwargs 吸收多余参数
   - 根因排查：通过在 _run_batch_to_tq 加临时 debug 日志打印 outcome 类型和 repr 定位

2. **容器名唯一化**
   - 原 run_id 为 f"swe_bb_{sample_index}"，n>1 时多个 session 使用相同容器名
   - 改为 f"swe_bb_{sample_index}_{uuid4().hex[:8]}"，每个 session 唯一
   - mini_swe_agent_runner 无此问题（DockerEnvironment 已自带 uuid）

3. **端口竞态修复**
   - LocalDeployment._pick_free_port() 使用 bind(0) 获取空闲端口后立即关闭 socket
   - 并发 session 可能拿到相同端口，导致 docker run -p 失败（exit status 125）
   - 修复：在 _create_agent_env 中预分配唯一端口，使用线程安全的 atomic counter
   - _port_counter 从 10000 递增，_port_lock 保证线程安全
   - 验证：重跑后端口冲突从多次降为 0 次

4. **SWE-bench 镜像 swerex 自动安装**
   - 部分镜像（seaborn、requests）未安装 swerex，导致容器启动后 swerex server 无法运行
   - sweb 镜像默认 python3 指向 conda env（Python 3.9），但 swerex 装在系统 Python 3.10
   - 修复：非 R2E 镜像的 command 改为先 pip install swe-rex 再用 python3.10 启动 swerex server
   - pip 包名为 "swe-rex"，模块名为 "swerex"
   - startup_timeout=600 足够覆盖 pip install 时间

5. **tool_parser 配置**
   - 训练模式从 actor_rollout_ref.rollout.multi_turn.format 读取 tool_parser
   - swe_agent_blackbox.yaml 缺少此字段，导致模型不调用工具（finish_reason=stop, tool_calls=None）
   - 添加 format: qwen3_coder
