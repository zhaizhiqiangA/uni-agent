核心修改内容（从上一次正式存档 archive_20260525_1700 至今）：

1. **删除 in-process reward bypass**
   - framework.py 的 _run_session 中，之前当 session_trajectories 有 reward_info 时跳过 _score_trajectories
   - 删除 has_in_process_reward 条件判断，有 reward_loop_worker_handles 时统一走 _score_trajectories → RewardLoopWorker → compute_score
   - SWEAgentFramework._score_trajectories 仍负责将 reward_info 合并到 extra_info，供 compute_score 读取

2. **推理模式接入 RewardLoopWorker**
   - parallel_infer.py 新增创建 RewardLoopWorker Ray actor：`ray.remote(RewardLoopWorker).remote(config, None)`
   - _init_hydra_config 中配置 reward.custom_reward_function 指向 compute_score，num_workers=1
   - parallel_infer.yaml 的 defaults 加入 `reward: reward`，引入 verl 标准 reward 配置（包含 reward_manager、sandbox_fusion 等 NaiveRewardManager 必需字段）

3. **run_train.sh Hydra 路径修复**
   - --config-path 从相对路径改为 "$(pwd)/examples/swe_agent_blackbox/config" 绝对路径
   - 原因：python3 -m verl.trainer.main_ppo_sync 执行时 Hydra 将相对路径解析为 verl/verl/trainer/ 下的子路径，导致找不到 config

4. **验证结果**
   - 推理（1 样本，uniagent runner，max_turns=2）：num_success_sessions=1，score 正确汇总，无 KeyError
   - 训练（Qwen3.5-4B，8 卡，2 步）：rollout → compute_log_prob → update_actor 全链路通过，checkpoint 保存成功
   - compute_score 通过 RewardLoopWorker 正常被调用，reward 分数正确传入训练流程
