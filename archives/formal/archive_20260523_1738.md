核心修改内容：

1. **MAX_TURNS 环境变量传递到 agent_runner**
   - agent_config.yaml 中 max_turns=100，但训练配置通过环境变量 MAX_TURNS=1
   - 环境变量只传给了 verl 的 multi_turn.max_assistant_turns，未传给 agent_runner
   - 导致 agent 实际跑了多轮交互（最多 7 轮），prompt 膨胀到 18k+ tokens 超出 max_model_len
   - 修复：agent_runner 读取 MAX_TURNS 环境变量覆盖 agent_config 的值
   - 验证：修复后 Exit due to max step limit: 1 正常触发

2. **compute_score 返回值格式修复**
   - verl 的 RewardLoopWorker.run_single() 期望 compute_score 返回 {"score": float} dict
   - 我们的 compute_score 返回裸 float，导致 RewardLoopWorker 解析失败
   - 修复：改为 return {"score": score}
   - 根因：verl 的 NaiveRewardManager.run_single() 检查 isinstance(result, dict) 后取 result["score"]

3. **_trajectory_to_reward_dataproto 缺失字段补默认值**
   - verl 标准数据集格式要求 data_source、reward_model（含 ground_truth）等字段
   - 我们的数据集（swe_local_8.parquet、r2e_local_8.parquet）只有 prompt、agent_name、extra_info
   - 缺少 data_source → RewardLoopWorker 报 KeyError('data_source')
   - 缺少 reward_model.ground_truth → RewardLoopWorker 报 KeyError('ground_truth')
   - 修复：在 _trajectory_to_reward_dataproto 中为缺失字段补默认值
   - data_source 默认 "unknown"，reward_model 默认 {"ground_truth": ""}

4. **纯文本输入跳过 VL position_ids 计算**
   - Qwen3.5-4B 的 processor 是 VL 版本，包含 get_rope_index 方法
   - 纯文本输入时 multi_modal_inputs 为空 dict，但 get_rope_index 仍被调用
   - get_rope_index 要求 mm_token_type_ids 参数，纯文本场景未提供 → TypeError
   - 修复：在 compute_position_ids 中，processor is None OR multi_modal_inputs 为空时
     直接走 compute_position_id_with_mask，跳过 VL 路径
   - 参考：verl 老架构 agent_loop.py 中 _compute_position_ids 有类似逻辑

5. **临时 debug 日志（待验证通过后移除）**
   - framework.py _run_prompt_sessions_to_tq 中添加 [TQ_DEBUG] 前缀日志
   - 记录每个 session 的成功/异常/空 trajectories 状态
