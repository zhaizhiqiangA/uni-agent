# 开发存档与回溯规则

本项目严格遵循以下存档与回溯机制，每次会话启动时必须读取并遵守。

## 1. 存档类型

| 类型 | 触发方式 | 用途 |
|------|----------|------|
| 临时断点存档 | 自动触发 | 中间变更留存，不作为最终版本 |
| 正式归档存档 | 用户指令【保存本次修改】 | 合并汇总生成最终归档，同时删除期间所有临时断点存档 |

## 2. 触发规则

### 2.1 自动临时断点（无需指令）
当识别到以下情况时，自动生成临时断点存档：
- 重要功能逻辑修改
- 关键配置变更
- 上下文压缩 / 会话截断即将发生

临时断点存档写入 `archives/temp/` 目录，文件名格式：`checkpoint_YYYYMMDD_HHMM.md`

### 2.2 手动正式归档（仅限用户指令）
只有用户发送【保存本次修改】时才执行正式归档：
- 把「上一次正式归档之后，所有自动临时断点 + 期间所有代码修改」合并汇总
- 生成一份正式归档文件
- 删除期间所有临时断点存档

正式归档写入 `archives/formal/` 目录，文件名格式：`archive_YYYYMMDD_HHMM.md`

## 3. 归档流程

正式归档按以下顺序执行（文档与代码一同提交，提交后再回填 commit 信息）：

### 3.1 同步更新文档 + 提交代码
1. **同步更新项目文档**：先更新 `DESIGN.md`（特性设计）和 `GUIDE.md`（执行指南）中与本次变更相关的内容，确保文档与代码保持一致
2. **提交 Git 记录**：将代码变更和文档变更放在同一个 commit 中一起提交，生成 7 位简短哈希

### 3.2 回填 commit 信息到归档
3. **生成结构化归档记录**：拿到 commit 哈希后，按固定格式写入归档文件（含 commit 号和描述）

## 4. Git Commit 规则

- commit 描述必须清晰写明：做了什么、解决了什么问题、优化了什么功能
- 统一使用 7 位简短 commit 哈希

## 5. 存档内容必填规则

- 记录范围：从上一次同类型存档到本次存档的全部核心修改
- 包含精确到年月日时分的时间戳
- 包含本次修改核心目的
- 罗列所有变更文件路径 + 每个文件关键变更说明
- 附带本次新提交 commit 的 7 位简短哈希

## 6. 正式归档固定输出格式

```
====================================
【开发存档】YYYY-MM-DD HH:MM
====================================
本次存档目的：
[本次开发的目标/解决的问题/优化点]

已修改文件列表：
- 文件路径1：变更说明1
- 文件路径2：变更说明2
- ...

Git Commit 信息：
commit 号：[7位简短哈希]
commit 描述：[清晰完整说明本次变更内容、解决问题、优化价值]

核心修改内容（从上一次正式存档至今，含所有自动临时断点变更合并汇总）：
[主要为文字描述，仅保留关键代码片段与逻辑说明，不要复制大段冗余代码]
====================================
```

## 7. 历史回溯支持

用户可随时下发以下指令：
- 列出所有正式归档记录
- 查看某一次正式归档完整内容
- 按 commit 号回溯版本
- 对比任意两次正式归档的差异

## 8. 目录结构

```
archives/
    temp/        # 临时断点存档
    formal/      # 正式归档存档
```

## 9. 会话启动自动读取

每次新会话启动时，必须执行以下步骤：
1. 读取 `.claude.md`（本文件）
2. 扫描 `archives/formal/` 目录，按文件名时间戳倒序排列
3. 读取最新一份正式归档的完整内容，作为本次会话的上下文基线
4. 若 `archives/temp/` 中存在未归档的临时断点，也一并读取

目的是让每个新会话都能无缝衔接上一次的开发进度，不丢失历史上下文。

## 10. 项目文档维护

每个开发目录下维护两份正式文档，正式归档时必须同步更新：

| 文档 | 内容 | 更新时机 |
|------|------|----------|
| `DESIGN.md` | 特性设计、架构决策、模块说明、依赖关系 | 架构变更、新增模块、设计决策变化时 |
| `GUIDE.md` | 执行指南、环境准备、数据集/权重准备、参数说明、故障排查 | 新增执行参数、流程变化、依赖变更时 |

- `examples/swe_agent_blackbox/docs/DESIGN.md` — 特性设计
- `examples/swe_agent_blackbox/docs/GUIDE.md` — 执行指南
- `examples/swe_agent_blackbox/docs/VERIFICATION.md` — 验证清单（状态跟踪）
